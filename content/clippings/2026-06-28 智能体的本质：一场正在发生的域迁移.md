# 智能体的本质：一场正在发生的域迁移

智能体的本质：一场正在发生的域迁移，以及你现在最应该做的事：

   
我们正在经历AI史上第二次域迁移。它不是产品升级，不是版本迭代，而是技术底层范式的更替。就像2012年之后手工特征工程不可避免地被淘汰一样，今天围绕"纯LLM"构建的工作方式，正在进入它的黄昏期。

   
这个判断可能让你不舒服。但我认为，不舒服的判断比舒服的模糊更有价值。

 

 
 
   
     
       一
     
     

## 什么是域迁移，为什么它比"技术趋势"严重得多

   

   
经济学家 W·布莱恩·阿瑟在《技术的本质》里提出了一个洞见：技术的演化，不总是在同一条轨道上加速，偶尔会发生底层切换——他称之为"域迁移"（redomaining）。

   
域，不是产品类别，不是市场赛道，而是围绕某种核心现象形成的一整套工具箱、运作原理和内在逻辑。真空管到半导体是域迁移，不是"更好的电子元件"；汽车不是"更快的马车"；蒸汽轮船不是"更大的帆船"。

   
域迁移有三个特征：

 

 
 
   
     
特征一

     
**旧域的工具在新域里只能凑合用**，因为底层范式不兼容。新域早期总是用旧域的零件拼装，看起来像"工程拼凑"，但那不是因为工程师不够聪明，而是新域还没有长出自己的原生工具。

   
   
     
特征二

     
**新域从母域中诞生，早期混乱，但最终会形成自己的核心技术**——那个"收拢一切的统一原理"。对深度学习来说，那个原理是"结构 + 损失函数 + 梯度下降"的铁三角。

   
   
     
特征三

     
**域迁移完成后，旧域不是消失，而是"隐没"**。晶体管没有消失，它变成了芯片里你不需要关心的基础设施。阿瑟说："新域会慢慢忘记它的母域，横空出世。"

        

用这个框架审视AI发展史，你会看到两次明确的域迁移，以及一次正在发生的。
      

     
       二
     
     

## 第一次域迁移的完整解剖

              

     
     
母域 · 1950s–2012

     
符号逻辑与手工特征工程时代

     
半个多世纪里，AI的主流范式是：专家定义规则，计算机执行规则。图像识别的工程师要亲手告诉机器"猫有耳朵、有胡须、有毛皮"，然后机器按照这些手工特征去匹配。这个范式有内在逻辑，有完整的工具箱，有自己的期刊和会议。它是一个成熟的域。

     
它的局限在于：现实世界太复杂，人类定义不了足够多的规则。它遇到了天花板。

   

   
   
     
     
触发点 · 2012年9月30日

     
AlexNet：范式证明

     
Krizhevsky提交的 AlexNet，在 ImageNet 挑战赛上以 **15.3%** 的错误率夺冠，第二名是 26.2%。差距之大，让评审以为数据有误。

     
但这不是参数调优的胜利。这是一次范式证明——机器不需要人类告诉它什么是特征，它能自己学。卷积层是眼睛，ReLU 是激活开关，反向传播是学习机制，GPU 是算力引擎。这四个要素第一次完整组合在一起，域迁移发生了。

   

   
   
     
     
混乱诞生期 · 2013–2016

     
框架战争

     
Theano 出身学界，是那个时代深度学习开发的行业标准，但设计偏底层，像研究平台而非工程工具。Caffe 由 UC Berkeley 博士生贾扬清创立，在计算机视觉领域最为流行，但架构问题让它不够灵活。Torch 被 Facebook 大力推广，却因为用 Lua 语言而让工程师望而却步。MXNet、CNTK 各据山头，没有共识。

     
这是域诞生期的典型症状：百花齐放，互不兼容，没有人知道哪条路是对的。工程师们在这堆碎片里拼凑答案，感觉混乱，但那不是因为方向错了，而是因为新域还没长出自己的骨架。

   

   
   
     
     
骨架收拢 · 2016–2019

     
TensorFlow 与 PyTorch

     
TensorFlow 带着谷歌的算力和工程体系强势入场，PyTorch 凭借动态计算图赢得研究圈。两强竞争最终让整个生态快速收敛，标准确立，生产工具成熟，Scaling Law 开始显现——更大的数据、更深的网络、更多的算力，带来可预测的性能提升。

   

   
   
     
     
成熟 · 2017至今

     
Transformer 统一一切

     
2017 年，《Attention Is All You Need》提出 Transformer 架构，用注意力机制把序列任务的所有结构收拢成一个统一框架。深度学习域进入成熟期：ChatGPT、GPT-4、Claude，都是在这个成熟框架上的产物。
       

LLM，是深度学习域的顶峰产物。**它同时也是智能体域的母域。 

 
 
   
     
       三
     
     

## 第二次域迁移：正在发生

   

   母域：LLM，预测下一个 token
   今天的大语言模型，核心范式是"预测下一个 token"。这是一台精密的概率机器，擅长模拟、生成、联想、翻译。它理解语言的方式是统计共现，它的内在逻辑是：下一个词最可能是什么？
   这个范式极其强大，但有一个根本性的边界：它没有目标，只有预测。它不"想"完成一件事，它只是在给定上下文里输出最可能的下一个词。

   新域的核心问题：如何让 AI 真正完成任务
 

 
 
   
     LLM 的问题
     下一个词是什么？
     相关性建模
   
   →
   
     智能体的问题
     为了达成目标，下一步行动是什么？
     效用建模
   
 

 
   前者是相关性建模**，后者是**效用建模**。这是两种不同的认知结构，底层范式不兼容。

   
这就是为什么今天的智能体工程如此痛苦：我们在用一个为"预测"设计的引擎，去驱动"规划与行动"——就像用照相机充当镜子，原理上行得通，但别扭是结构性的。

   
混乱的诞生期：我们现在正在这里

   
你打开今天任何一个智能体框架的文档，你会看到： 

 
 
   
     ⚙️
     
精心调教的 System Prompt，像咒语一样

   
   
     🔗
     
手动编排的工具调用链，像流水线工人

   
   
     🩹
     
各种补丁式的记忆方案，试图解决模型失忆的问题

   
   
     ⚔️
     
LangGraph、CrewAI、AutoGen……框架战争正在上演

   
        
这和 2013–2016 年的深度学习框架混战，几乎一模一样。这不是工程师不够聪明，这是域迁移诞生期的必然形态：新域还没长出自己的原生工具，只能用旧域的零件拼凑。     

       四
     
     

## 三大结构性矛盾，解释了为什么今天"感觉很难"

   
 

 
 
   
     
矛盾一

     
认知范式的错位

   
   
     
深度学习域的核心假设：有足够多的数据，模型能自己学出规律。

     
智能体域的核心需求：在没有标准答案的开放环境里，模型能自主规划并执行多步任务。

     
这两个范式，不是深浅之别，是方向之别。你没办法用"预测"的范式无缝切换到"规划"的范式，就像你没办法用统计语言学的工具无缝切换到神经网络——你需要新的核心技术。   

 
 
   
     
矛盾二

     
"反向传播等价物"——这个问题正在被解决

   
   
     
深度学习之所以能指数进化，根本原因是反向传播：每一次失败，误差信号精准地反传到每一个权重，模型真实地改变自己。

     
"智能体没有反向传播，所以无法从失败中学习。"——这个说法在 2022 年准确，在 2026 年已经过时了。

     
       
关键突破 · 2025年1月20日

       
DeepSeek-R1 发布，第一个在推理能力上达到 OpenAI o1 水平的开源模型。其核心机制 **RLVR（可验证奖励强化学习）**证明了：可以不依赖人工标注的推理轨迹，仅靠强化学习，让模型发展出复杂的推理能力——这正是"智能体的反向传播"的早期形态。

     
     
真正仍然悬而未决的问题不是"有没有反向传播"，而是更难的那个：**如何为开放域任务设计泛化的奖励函数**。数学题有标准答案，代码有测试用例——但"帮我完成这次商业谈判"，怎么定义"完成"？这是智能体域当前最核心的未解难题。  

 

 
 
   
     
矛盾三

     
世界模型的缺失——但它正在加速到来

   
   
     
神经网络的世界是可微分的张量。智能体面对的现实是离散的 API 调用、不确定的环境反馈、非线性的任务结果。要真正解决这个矛盾，需要一种能够把"现实世界"建模为可微分内部空间的系统——世界模型。

     
2025 年，世界模型领域迎来密集突破：

     
       
       
**Google DeepMind · Genie 3**：首个能实时以 24 帧/秒、720p 分辨率生成可交互环境的通用世界模型

     
     
       
       
**Meta · V-JEPA 2**：在超过百万小时的互联网视频上训练后，能零样本控制真实机械臂

     
     
       
       
**NVIDIA · Cosmos**：面向物理 AI 的开源世界基础模型平台

     
     
       
最大单笔赌注 · 2026年初

       
Yann LeCun 离开 Meta，创立 AMI Labs，以 30 亿欧元估值专注于世界模型研究——这是目前对"超越 LLM 的 AI 路径"的最大单笔赌注。
      
 

 
 
   
     
       五
     
     

## 智能体域的"骨架"：什么已经收拢，什么还缺失

   
 

 
 
   
     深度学习域
     智能体域对应物
     状态
   
   
     卷积核 / Transformer
     推理与规划循环（ReAct等）
     碎片化
   
   
     批归一化
     上下文管理与记忆系统
     未收敛
   
   
     ReLU 激活函数
     MCP 工具调用协议
     已收拢 ✓
   
   
     框架标准（PyTorch/TF）
     LangGraph / CrewAI / SDK
     竞争中
   
   
     反向传播
     RLVR / 过程奖励模型（PRM）
     突破中
   
   
     统一损失函数
     开放域泛化奖励设计
     最大空白
   
   
     训练环境（ImageNet等）
     世界模型（Genie 3 / Cosmos）
     建设中
   
 

 
   
MCP 协议的收拢值得单独说一句。到 2025 年夏天，MCP 作为智能体连接外部世界的主流协议地位已经确立，OpenAI、Google 相继采纳，Salesforce 在 Agentforce 3 中将其作为核心互操作基础。协议战打完了，上层建设会骤然加速。

   
     
       六
     
     

## 域迁移的生命周期：我们在哪里，接下来会怎样

   
 

 
 
   
     
       
诞生期

       2024–2026 · 我们在这里
     
     
所有人都在用旧域（LLM）的零件拼凑新域的产品，Prompt 工程主导，框架竞争激烈，没有公认的最佳实践，大量方案"能用但不够好"。

     
这个阶段最容易让人产生一种错觉：这个方向是不是根本走不通？**不。**这不是走不通，这是新域诞生期的正常形态。

   
   
     
       
青春期

       2026–2030
     
     
新域开始产生自己的原生工具，可以预期的关键突破：

     
       ▪
       
**原生架构**：出现专门为"行动序列"设计的基础模型，从训练目标、数据结构、推理机制上，都为"规划与执行"而生

     
     
       ▪
       
**可微分奖励**：世界模型成熟后，开放域任务的"完成度"第一次能被转化为可训练的数学目标

     
           ▪
       
**记忆固化**：真正的跨任务学习出现——智能体不再是每次对话都从零开始的"金鱼"

     
   
   
     
       
成熟期

       2030+
     
     
App 生态被 Agent 生态取代，不是因为有人宣布"App 时代结束了"，而是因为在越来越多的任务场景里，构建一个 Agent 比构建一个 App 更快、更便宜、更有效。人们不再谈论 LLM，就像今天的人不谈论晶体管——它已经隐没在底层，成为不可见的基础设施。

   
 

 
 
   
     
       七
     
     

## 那个"AlexNet 时刻"，不会是你想象的样子

   

   
关于智能体域的下一个共识预期是：某一天，某篇论文会给出一个"统一智能体域"的架构，就像 Transformer 统一了 NLP 一样，然后一切指数爆发。

   
我不认为它会是这样的。

   
回溯 2012 年：没有人在那一天意识到历史转折发生了。Krizhevsky 和 Hinton 感受到的，是"数字好得令人困惑"。之后的一切——CNN 的统治、GPU 算力的疯狂投入、深度学习博士的薪资暴涨——是在接下来三年里慢慢清晰的。

   
智能体域的临界点，可能是这样一个不起眼的时刻：

 

 
 
   
某家律所的研究智能体，把案件准备时间从 **40 小时**压到 **4 小时**，然后这件事在圈子里扩散。

   
某个跨国供应链，第一次在无人监督的情况下独立处理了一次突发中断。

   
某个产品团队发现，他们的智能体处理客服工单的准确率，已经连续六个月超过 90%。

 

 
   
这些事情不会上头条。但它们会出现在季度财报的"运营效率"一栏。

   
等大多数人意识到"智能体时代来了"的时候，先动手的人已经积累了 18 个月的失败经验和可复用的工作流。  

## 尾声：域迁移早期，永远长得像工程拼凑

   
最后一句话，留给那些此刻感到困惑的人。

   
你打开智能体框架的文档，看见密密麻麻的 Prompt 模板、工具调用配置、记忆系统搭建……然后心里升起一个疑问：就这？这就是所谓的"智能体时代"？

   
这个感觉，和 2014 年的深度学习工程师如出一辙。他们用 Theano 和 Caffe 拼凑模型，看着那些繁琐的配置，也会想：就这？

 

 
 
   
域迁移的早期，
永远长得像工程拼凑。

   
 

 
   
因为新域的核心技术还没有收拢，所有人都在用旧域的零件搭积木。那些积木，既能用，又不完美，既让你看见可能性，又让你感到别扭。

   
这种别扭感，不是在告诉你这条路是错的。

   
它在告诉你，你正处于对的地方，对的时间。

   
智能体域的 Conv、BatchNorm、ReLU 已经散落在桌上：MCP 协议、RLVR 训练机制、世界模型、记忆系统。它们都存在了。差的，是那个把它们端到端串联起来的统一原理——那个"任务完成"的可微分损失函数，以及那个让智能体真正能"通过行动改变自身"的训练范式。

   
那个东西，一旦出现，将是无声的。

   
没有头条，没有发布会，可能只是某个实验室的某篇论文，某个指标曲线突然开始持续下降。 

 
 然后，十年之后，我们会翻出那篇论文，说：   
就是这里，那个 AlexNet 时刻。
