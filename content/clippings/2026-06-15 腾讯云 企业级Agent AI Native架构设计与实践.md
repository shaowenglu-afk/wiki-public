# 企业级 Agent AI Native 架构设计与实践

> 内容源自 ITPUB【中国系统架构师大会】主题分享。

---

## 5 个核心观点（开场金句）

1. **模型能力驱动智能体架构发展**——没有成熟的智能体架构，只有不断精进的智能体架构
2. **工程化思维**：除传统 SDD/TDD/DDD 外，Agent 时代应更加注重 **EDD（Eval-Driven Development）**
3. 企业的智能体建设，未来应该朝着 **APaaS（Agent Platform as a Service）** 方向去做
4. **Markdown 设计文档即代码** —— 企业必须从 Vibe Coding 转向 AI 规范化编程，拒绝 "shi 山" 代码
5. **现在企业级 Agent AI Native 架构跑不出 7+2 结构**：
   - **7 层架构**：业务应用层 / 应用网关层 / AI AGENT CORE / AI 网关层 / 大模型层 / 知识 & 数据层 / 基础设施层
   - **2 侧横切关注点**：可观测性 & 评估 / 安全治理 & 合规

---

## 一、重新认识 AI Native 架构

### AI 集成 vs AI Native（关键定义）

| 维度 | AI 功能集成 | AI Native |
|---|---|---|
| 主体 | 系统本身（AI 是外挂模块）| AI 大模型为核心 |
| 模型调用 | 无状态 | 驱动核心业务流程 |
| 一句话 | **「给系统加 AI 功能」** | **「用 AI 重构系统」** |

### 传统智能体 vs 企业级智能体

- **传统智能体**：解决能力**有无**问题，整体 Harness 偏弱，常是 MVP 阶段产物
- **企业级智能体**：要求**高效、稳定、安全**——任务复杂度、记忆体系、安全性、权限控制要求**远高于**传统智能体

### 企业级智能体落地的三次关键转变（行业范式）

1. **从上下文工程 → 驾驭工程**
2. **从单体智能体 → 多智能体**
3. **从项目级智能体 → 平台级智能体**

**核心解决三大问题**：可控性不足、可扩展性不足、可治理性不足

### Agent = Model + Harness（核心公式）

> 智能体架构是在持续精进，**每次换模型时，Harness 必须重新评估一次**——因为随着模型能力增强，**有些原本 Harness 上需要关注的技术点，都会被模型吞噬**。

## 二、企业级 Agent AI Native 架构（7+2）

### 7 层架构

| 层 | 名称 | 核心职责 |
|---|---|---|
| 1 | **业务应用层** | 与智能体交互的终端（Chatbot / plugin） |
| 2 | **应用网关层** | 传统应用网关，负载均衡 / 路由 / 鉴权 |
| 3 | **Agent 核心层** ⭐ | 智能体框架、构建方式、记忆体系设计 |
| 4 | **AI 网关层** | 对大模型调用统一管理：**语义路由 / 语义缓存 / Token 配额** |
| 5 | **大模型层** | 各类大模型集中管理 |
| 6 | **知识 & 数据层** | 结构化 + 非结构化数据，知识库 |
| 7 | **基础设施层** | 智能体运行所需的底层能力 |

### 2 侧横切关注点（贯穿 7 层）

| 侧 | 关注点 |
|---|---|
| **左侧**：可观测性 & 评估 | 传统链路追踪 + 大模型专属指标（**TTFT / Token 用量**）+ 离线评估 + 在线评估 |
| **右侧**：安全治理 & 合规 | 输出可控、不越界、符合企业安全合规要求 |

## 三、核心层设计与技术选型

### Prompt → Context → Agent Runtime（演进）

三者是**层层递进的包含关系**：
- **Prompt** 写得越来越少
- **Context** 组装至关重要—— 让模型在此刻看到应该看到的内容，**既不多，也不少**
- **Harness** 优化的是**系统工程**

### 4 层记忆体系

| 层 | 名称 | 性质 | 存储 |
|---|---|---|---|
| 1 | **工作记忆**（短期） | 当前会话中任务的中间状态、工具返回值、推理步骤 | 内存 / 关系型 DB / KV |
| 2 | **情景记忆**（长期） | 跨会话持久化：历史对话、用户偏好、任务轨迹 | 关系型 DB / NoSQL / 向量库 |
| 3 | **语义记忆**（长期） | 知识库 | 向量库 + 知识图谱 |
| 4 | **程序性记忆**（长期） | 偏"怎么做"：SOP / Workflow / Skill 定义 —— **经验沉淀成可执行能力** | — |

### 知识库演进（RAG → Agentic RAG）

去年的重大飞跃：**从传统 RAG（Naive / Advanced）进化为 Agentic RAG**——
- **本质**：从固定 Pipeline → 动态决策检索
- **常配合使用**：让知识库检索更灵活
- **大量实体关系**：考虑引入知识图谱增强推理能力

## 关键启示

1. **AI Native ≠ AI 集成**——一个是加，一个是重构
2. **7+2 架构**是中国语境企业级 Agent 落地的明确范式
3. **Agent = Model + Harness 公式**——模型增强会蚕食 Harness（与 [[concepts/Harness-Engineering]] 呼应）
4. **EDD（Eval-Driven Development）** 是 Agent 时代的工程化新支柱
5. **APaaS（Agent Platform as a Service）** 是企业未来方向
6. **4 层记忆体系**清晰可工程化：工作 / 情景 / 语义 / 程序性
7. **Agentic RAG** 替代 Naive RAG 是 2026 知识库主流路线
