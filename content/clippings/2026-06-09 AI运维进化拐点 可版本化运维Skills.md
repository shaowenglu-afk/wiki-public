# AI 运维进化拐点：可版本化运维 Skills

## 中文 TL;DR · 极具启发性

**核心反共识**：AIOps 的能力齐全但章法缺失。**比大模型更重要的，是把人的经验沉淀成可版本化、可复用、可迭代的运维 Skills**——类似 Kubernetes CRD / 模块化代码。

## 7 章纲要

### 一、当下 AIOps 的普遍困境：能力齐全，章法缺失
- 各家都有 LLM + Agent + 工具，但**每次都重新发明轮子**
- 经验在脑子里，新人来 = 从零开始

### 二、Skills 核心解法：给 AI 运维装上标准化经验包
- **Skill = 一个可执行单元 + Prompt + 工具 + 验收标准 + 历史成功样本**
- 类似 Kubernetes CRD —— 声明式、可版本化

### 三、Skills 极简落地流程：3 步标准化 AI 运维

### 四、三大开箱即用 Skill，覆盖核心运维场景
- 例如：服务重启 Skill、慢查询 RCA Skill、容量预测 Skill

### 五、不止是 Prompt 优化，更是平台级 AI 运维升级
- Skill 是**平台资产**，可被多个 Agent 调用

### 六、持续迭代进化，打造可生长的运维技能生态
- Skill marketplace / Skill 版本控制 / Skill 性能监控

### 七、结语：AI 运维的分水岭，是标准化经验沉淀

## 关键论点 · "Skills as Code"

- ⭐ **运维经验 = 可版本化数字资产**（不再藏在工程师脑子里）
- ⭐ **Skill 是 AI Agent 的"标准操作手册"**
- ⭐ **Skill marketplace = 运维知识的开源仓库**
- 与 Anthropic Skills、Claude Code Skills 同构思想

## 对运维人员的启示

| 老路 | 新路 |
|---|---|
| 个人英雄主义 + 经验在脑里 | **经验沉淀成 Skill** |
| 重复救火 | 救完火**写 Skill**，下次 AI 自动跑 |
| 个人能力 = 私有资产 | **个人能力 = 平台资产**（影响力更大）|
