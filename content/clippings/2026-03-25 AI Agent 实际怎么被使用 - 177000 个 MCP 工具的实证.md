## 摘要

本研究考察配备工具的 AI Agent 如何访问和修改外部环境。研究指出：

> "**开发者、研究者和政府都缺乏对 AI Agent 当前如何被用于关键任务的理解**。"

## 方法

研究者评估了 **2024-11 到 2026-02** 之间创建的 **177,436 个 Agent 工具**——监控**公开的 MCP（Model Context Protocol）服务器仓库**。

工具按影响类型分类：
- **感知（perception）**：数据访问/读取
- **推理（reasoning）**：数据分析
- **行动（action）**：环境修改

用 **O*NET 数据库** 把工具映射到任务领域，评估后果性（consequentiality）。

## 关键发现

- **软件开发**代表 **67% 的所有工具**和 **90% 的下载量**
- **行动类工具**显著增长：在 16 个月期间内**从 27% 上升到 65%**
- 大多数行动工具支持**中等风险任务**（文件编辑、邮件）
- 高风险应用已经出现，**包括金融交易能力**
- 研究展示了**关注"工具层"而非只看模型输出**的监管监测方法

## 数据源

公开的 Model Context Protocol 服务器仓库，从 2024 年末追踪至 2026 年初。

## 意义

- 证实了 **MCP 已成为 AI Agent 工具生态的事实标准基础设施**
- 验证了**软件开发是 Agent 工具最主要的下游应用**
- 揭示了从"读"到"做"的快速跃迁——这是 Agent 风险评估的关键信号
- 为治理提供新视角：**不只看模型，看工具层**
