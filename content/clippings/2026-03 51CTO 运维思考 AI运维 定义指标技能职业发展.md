# 运维思考：AI 运维解析

## 中文 TL;DR · 国内一线运维视角

国内 51CTO 资深运维工程师的系统化思考——AIOps 从定义到职业发展路径的完整框架。

## 4 大主题

### 1. AI 运维定义
- 不只是"加个 AI 模块"
- 是**用 AI 重构整个运维流程**

### 2. 关键指标
- **MTTR**（平均修复时间）—— 行业基线 -70%
- **告警去噪率** 95%+
- **自动化率** —— 多少比例可自动处理

### 3. 必备技能储备 ⭐
**5 大必备**：
1. **传统运维基本功**（K8s / Linux / 网络）—— 没基础学 AI 也是空中楼阁
2. **可观测三件套**（Prometheus / Grafana / Loki / Tempo）
3. **Python + 简单 ML**
4. **LLM + Prompt 工程**
5. **Agent 框架**（LangGraph / Claude Agent SDK）

### 4. 职业发展路径

| 阶段 | 头衔 | 关键能力 |
|---|---|---|
| 起步 | 传统运维工程师 | 救火能力 |
| **过渡** | **AIOps Engineer** | **传统运维 + AI 应用** |
| **进阶** | **AI SRE Architect** | **多 Agent 架构设计 + 治理** |
| 顶级 | Platform Lead | 平台战略 + 跨团队 |
