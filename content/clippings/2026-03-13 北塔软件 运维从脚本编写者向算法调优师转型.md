# 北塔：运维从脚本编写者向算法调优师转型

## 中文 TL;DR

**国内 IT 运维管理软件厂商北塔**视角：定义了运维角色 4 大演进阶段：

## 4 大演进阶段

| 阶段 | 角色 | 代表能力 |
|---|---|---|
| 1.0 | **手工运维** | 命令行操作 |
| 2.0 | **脚本编写者** | Shell / Python / Ansible |
| 3.0 | **平台架构师** | IaC / GitOps / K8s |
| **4.0** | **算法调优师** ⭐ | 调 AI 模型阈值 + 优化 Agent 推理 + Skill 设计 |

## 关键论点
- **脚本逻辑 → 算法逻辑** 的根本转变
- 调参从 Nginx config → AI 模型阈值
- 写 if-else → 写 Prompt + 验收标准

## 国内厂商视角的价值
北塔作为国内 IT 运维管理软件厂商（vs 国外 Datadog / Dynatrace），看到的是**中国企业实际转型现状**——很多企业还在 2.0 → 3.0 过渡，**直接跳到 4.0 难度大**。
