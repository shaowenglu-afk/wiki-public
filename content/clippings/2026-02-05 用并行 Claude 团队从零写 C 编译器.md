## 概览

Anthropic 研究员 Nicholas Carlini 描述了一个用 "**Agent 团队**" 的实验——多个 Claude 实例在共享代码库上**并行工作、无人主动干预**。他给 **16 个 Agent** 派了一个任务：从零写一个基于 Rust 的 C 编译器，要能从头编译 Linux 内核。

## 项目结果

Agent 团队产出了一个 **10 万行**的编译器，跨越近 **2000 次 Claude Code 会话**，成本约 **$20,000**。这个编译器：

- 成功编译 **Linux 6.9**，覆盖 **x86 / ARM / RISC-V** 三种架构
- 能编译 **QEMU、FFmpeg、SQLite、PostgreSQL、Redis**
- 通过 **99% 的编译器测试套件**

## 技术实现

### 长任务 Agent

Carlini 创建了一个让 Claude 自主工作的循环系统。Agent 之间不等待人类输入，**做完一个任务立即拿下一个**：

> "When it finishes one task, it immediately picks up the next."

### 并行处理

多个 Docker 容器同时运行，共享一个 git 仓库。Agent 之间通过 `current_tasks/` 目录下的文本文件锁防止冲突。当两个 Agent 试图做相同的工作时，**git 同步会强制其中一个选择不同的任务**。

## 关键经验

### 测试质量是一切之最

高质量测试至关重要。Carlini 强调：

> "**任务验证器必须接近完美，否则 Claude 会去解决错误的问题。**"

他构建了持续集成流水线，并针对 Claude 遇到的具体失败模式定制测试。

### 为模型设计

不是按"人会怎么测"来设计，而是**针对 Claude 的能力和局限优化环境**：

- 维护详尽文档
- 避免上下文污染
- 加入确定性采样选项防止测试无限循环

### 启用并行化

当 Linux 内核成为单一巨型任务时，Carlini 把 GCC 当 "**oracle**"——随机让 GCC 编译一部分、让 Claude 的编译器编译另一部分。这让 Agent 可以独立工作于不同文件。

### 专门化角色

除核心编译器开发外，专门化的 Agent 负责：
- 代码去重
- 性能优化
- 代码质量分析
- 设计批评（design criticism）
- 文档

## 项目局限

完成的编译器有明显约束：

- 缺少 16-bit x86 代码生成器，这一阶段交给 GCC
- 缺少自定义汇编器和链接器（仍有 bug）
- 不能成功编译所有项目
- 生成代码效率落后于 GCC
- Rust 代码质量低于专家水平

Carlini 指出 16-bit x86 生成器特别难——输出大小**几乎是 32KB 限制的两倍**。

## 性能数据

- 消耗 **20 亿 输入 token + 1.4 亿 输出 token**
- 两周内近 2000 次 Claude Code 会话
- **Clean-room 实现**，只依赖 Rust 标准库
- 通过 torture test 套件，能编译 Doom

## 更广泛的启示

Carlini 反思从传统 LLM 交互模式向自主开发的转变。早期范式是简短的模型调用 + 即时人类反馈；Agent 团队则代表**持续自主工作**的转向。他承认这带来风险：

> "程序员部署他们从未亲自验证过的软件——这是个真实的担忧。"

快速进展表明，以前被视为遥远的能力正在**比预期更快地兑现**，需要新的安全和验证框架来跟上。
